# Reference Docs

This folder contains supporting historical notes. It is not the active architecture.

Read `docs/openvision/00_INDEX.md` first. Interpret every reference here through the current OpenVision rules:

- Rokid stays thin;
- Jetson owns realtime perception, skill runtime, and HUD authority;
- cloud AI is typed escalation;
- every user-visible visual result goes through HUD scene protocol;
- v1 modes and one-off routes are not product direction.

Useful reference categories:

- RV101 display/HUD constraints;
- audio protocol lessons;
- browser/iPhone media notes;
- old OpenAI Realtime experiments;
- field notes that may still inform implementation.

Do not treat these files as permission to revive old modes, direct cloud calls, hidden STT routing, or legacy backend structure.
